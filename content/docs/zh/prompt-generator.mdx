## 功能概述

提示词生成器用于帮助你更快写出结构化的系统提示词。  
它最适合做“起稿”和“优化”，而不是完全替代人工判断。

## 适用场景

适合：

- 第一次创建 Agent，不知道如何组织提示词
- 现有提示词效果不稳定，需要重新梳理边界
- 希望快速生成不同风格的草稿版本

## 前置条件

开始前建议准备：

- Agent 的目标描述
- 目标用户和典型任务
- 你希望它遵守的输出格式和限制条件

## 操作步骤

### 第 1 步：先输入真实需求，而不是空泛目标

需求越具体，生成结果越可用。  
应尽量明确：

- 角色
- 任务
- 输出格式
- 禁止事项

### 第 2 步：用生成结果作为草稿

把生成结果看成第一版，而不是最终版。  
重点人工补充：

- 业务边界
- 错误处理方式
- 术语和口径

### 第 3 步：回到 Agent 中做真实验证

生成后必须放回实际 Agent 里测试，观察：

- 输出是否稳定
- 是否过度自信
- 是否能在信息不足时正确收口

## 结果验证

提示词生成器真正有价值时，应表现为：

- 起稿更快
- 结构更完整
- 迭代时更容易比较版本差异

## 价值说明

它的价值在于降低写提示词的起步门槛，但最终质量仍依赖人工验证和业务理解。

## 注意事项

- 不要直接把生成结果原样上线
- 对高风险场景，必须人工补足边界和禁止规则
- 最终评判标准始终是实际测试结果，而不是提示词看起来多完整
